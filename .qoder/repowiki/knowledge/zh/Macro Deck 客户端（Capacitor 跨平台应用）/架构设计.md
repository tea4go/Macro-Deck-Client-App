- **单仓多产物**：`src/` 下的 Angular/Ionic 代码经 `ionic build` 产出静态资源，再由 `capacitor.config.ts` 注入 `android/` 与 `ios/` 两个原生壳工程，形成“一次开发、三端运行”的混合架构。
- **CI 编排契约**：根级 `.github/workflows/ci.yml` 作为总入口，通过 `reusable-determine-version.yml` 计算语义化版本，再并行调用 `reusable-base-build.yml`（Web 构建）、`reusable-android-*` 与 `reusable-ios-*` 生成各平台产物，保证版本号在多端严格一致。
- **配置分层**：`angular.json` 定义 Web 构建目标与环境变量，`ionic.config.json` 与 `package.json` 协调 CLI 行为；Android 与 iOS 各自维护独立的 `fastlane/Fastfile` 完成签名与分发，脚本侧由 `scripts/` 提供 macOS/Windows 双栈封装。
