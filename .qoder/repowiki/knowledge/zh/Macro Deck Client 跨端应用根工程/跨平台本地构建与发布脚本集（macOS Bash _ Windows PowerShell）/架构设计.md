按操作系统切分为三个互不交叉引用的目录：
- `macos/_common.sh` 作为共享库，通过 `source _common.sh` 被各入口脚本加载，集中实现日志（write_ok/warn/fail/banner）、环境预检（require_command/require_env/test_nodejs）、构建编排（prepare_web_build → ionic build -c production → cap sync <platform>）、fastlane 命令选择（优先 bundle exec fastlane）等公共能力。
- `windows/_common.ps1` 以相同语义封装 PowerShell 侧等价函数（Write-Ok/Warn/Fail/Banner、Confirm-* 交互确认、Invoke-NativeStream 原生命令包装），并额外提供 Windows 专属工具链适配：NTFS junction 路径解析（Resolve-JunctionPath）、并行测速下载（Save-WebFile 先 Start-Job 多源竞速再单线程稳定下载）、Gradle wrapper 镜像替换（腾讯云）、jvms JDK 版本切换（从注册表刷新 PATH/JAVA_HOME）、Android SDK 多候选根探测（ANDROID_HOME/ANDROID_SDK_ROOT/LOCALAPPDATA/项目约定路径）。
- `local/` 仅存放敏感配置模板（android-signing.env.example / android-signing.ps1.example），禁止提交至版本控制；CI 通过仓库 Secrets 注入同名环境变量覆盖本地值，两套来源在运行时合并到同一组 KEYSTORE_FILE_PASSWORD/BUILD_NUMBER/VERSION_NUMBER 变量中。
- 每个平台内脚本职责单一且编号有序：install_1_base_tools → install_2_ruby → install_3_fastlane → install_4_android_sdk；remove_* 反向卸载；build_* 执行 Web→Capacitor sync→原生打包；run_* 启动开发服务器或模拟器；Windows 侧额外提供 sync_version_bywin.ps1 同步三端版本号。
