- **单仓三产物**：`src/` 下的 Angular/Ionic 代码经 `ionic build` 产出静态资源，再由 `capacitor.config.ts` 同步到 `android/`、`ios/` 两个原生壳工程，形成“一次开发、三端运行”的混合架构。
- **CI 编排契约**：根级 `.github/workflows/ci.yml` 作为总入口，先调用 `reusable-determine-version.yml` 计算语义化版本，再并行触发 `reusable-base-build.yml`（Web）、`reusable-android-*` 与 `reusable-ios-*`，确保版本号在多端严格一致。
- **配置分层**：`angular.json` 定义 Web 构建目标与环境变量，`ionic.config.json` 与 `package.json` 协调 CLI 行为；Android 与 iOS 各自维护独立 `fastlane/Fastfile` 完成签名与分发，脚本侧由 `scripts/` 提供 macOS (Bash) / Windows (PowerShell) 双栈封装。
