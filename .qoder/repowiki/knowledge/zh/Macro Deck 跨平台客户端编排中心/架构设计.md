- **核心桥接机制**：以 `capacitor.config.ts` 为中枢契约，将 Angular/Ionic Web 应用（输出至 `www`）注入 Android 与 iOS 原生壳工程，实现“一次开发，多端原生运行”的架构。
- **流水线编排**：根目录 `.github/workflows/ci.yml` 作为总控入口，通过 `workflow_call` 串联版本判定、Web 基础构建以及 Android/iOS 并行原生编译，确保多平台产物版本严格一致。
- **配置统一管理**：通过 `angular.json` 定义多环境（Web/Production）构建策略，并结合根级 `package.json` 与 `ionic.config.json` 协调 CLI 工具链在开发与发布阶段的统一行为。
