- 目录按平台切分：`macos/` 下 Bash 脚本以 `_common.sh` 为共享库，`windows/` 下 PowerShell 脚本以 `_common.ps1` 为共享库，`local/` 仅存放敏感配置模板（`.env.example` / `.ps1.example`），三者互不交叉引用。
- 每个平台内脚本职责单一：`install_N_*` 顺序安装 Ruby/fastlane/Android SDK；`build_*` 执行 Web→Capacitor sync→原生打包；`run_*` 启动开发服务器或模拟器；`remove_N_*` 反向卸载；`sync_version_bywin.ps1` 同步三端版本号。
- 入口脚本通过 `source _common.sh` 或 `. (Join-Path $PSScriptRoot '_common.ps1')` 加载公共函数后，统一走“环境预检 → 变量装载 → 调用 npm/ionic/capacitor/fastlane”的执行顺序，并遵循 `prepare_web_build → ionic build -c production → cap sync <platform>` 的标准流水线。
- 敏感信息隔离：Android 签名密钥通过 `scripts/local/android-signing.env`（macOS）或 `scripts/local/android-signing.ps1`（Windows）加载，GitHub Actions 则用仓库 Secrets 注入同名环境变量，两套来源在运行时合并到同一组 `KEYSTORE_FILE_PASSWORD`/`BUILD_NUMBER`/`VERSION_NUMBER` 变量中。
