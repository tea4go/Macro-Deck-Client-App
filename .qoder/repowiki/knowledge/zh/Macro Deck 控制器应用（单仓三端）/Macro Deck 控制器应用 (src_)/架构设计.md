以 `src/app` 为应用边界，按领域分层组织：`datatypes/` 与 `enums/` 提供跨子模块共享的数据契约；`services/` 封装从底层通信（`WebSocketService`）到协议层（`Protocol2Service`/`ProtocolHandlerService`）、硬件适配（`ScreenOrientation`、`WakeLock`、`CurrentPlatform`）再到业务编排（`ConnectionService`、`MacroDeckService`）的依赖链；`pages/` 下的 Home、Deck、WebHome 等页面通过懒加载 Module 按需引入服务；`widget-content-components/` 作为可插拔的微件 UI 插件，由 `WidgetContentComponent` 根据 `WidgetContentType` 动态实例化。根入口 `main.ts` → `AppModule` → `AppComponent` 完成全局初始化（i18n、主题、平台检测、深度链接），并通过 `NavigationService` 驱动页面路由，形成「UI 组件 → Service → Protocol → WebSocket」的单向数据流。
