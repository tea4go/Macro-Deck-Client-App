- 入口层：`app/src/main/java/com/suchbyte/macrodeck/MainActivity` 继承 `BridgeActivity`，在 `onCreate` 和 `onWindowFocusChanged` 中通过 `SYSTEM_UI_FLAG_IMMERSIVE_STICKY` 等标志位强制维持全屏沉浸式体验，并在系统 UI 可见性变化时自动恢复。
- 资源与 Web 宿主：`assets/public/` 内嵌由前端构建产物（Angular + PWA）生成的 JS/CSS/SVG 及 `ngsw-worker.js`；`capacitor.config.json` / `capacitor.plugins.json` 声明 WebView 配置与插件映射；`res/xml/config.xml` 为 Cordova 兼容配置。
- 多模块 Gradle 结构：根 `build.gradle` 引入 `variables.gradle` 集中声明 SDK 与依赖版本（compileSdk/targetSdk/minSdk、AndroidX、Cordova Android），子模块 `app/build.gradle` 动态检测 `google-services.json` 以决定是否启用 Google Services 推送插件；`capacitor-cordova-android-plugins/` 作为独立模块承载 `CDVOrientation` 等 Cordova 插件源码。
- 自动化层：`fastlane/Fastfile` 定义 `build` 与 `release` 两条车道——前者校验 `BUILD_NUMBER`/`VERSION_NUMBER` 环境变量后调用 `increment_version_code/name` 修改 `app/build.gradle`，再依次执行 `bundle`(AAB) 与 `assemble`(APK)，并重命名 AAB 输出；后者通过 `upload_to_play_store` 将 AAB 以 draft 状态上传至 Play Console。
- 依赖方向：`MainActivity` 仅依赖 Capacitor `BridgeActivity` 与 Android SDK，不直接引用业务逻辑；Web 页面通过 Capacitor 插件 API 访问原生能力，形成单向桥接。
