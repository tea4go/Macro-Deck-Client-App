## 简介
本文件系统性梳理 Macro-Deck-Client-App 的构建配置，覆盖 Angular CLI 配置、TypeScript 编译配置、环境变量、Service Worker 配置与构建优化策略，并结合实际源码进行可视化说明，帮助开发者快速理解与高效维护构建流程。本次更新重点介绍了新增的 Web/PWA 构建配置、PowerShell 构建脚本集成以及完整的构建优化策略。

## 项目结构
该工程采用 Angular + Capacitor + Ionic 的混合架构：
- 使用 Angular CLI 进行浏览器端与原生打包
- 使用 Capacitor 将 Web 资源桥接到原生平台
- 使用 Ionic 提供 UI 组件与主题样式
- 通过 Service Worker 实现 PWA 缓存策略
- 通过 PowerShell 脚本实现自动化构建流程

## PowerShell 构建脚本集成

### 脚本概述
PowerShell 构建脚本提供了完整的 Web/PWA 构建自动化流程，支持环境检查、依赖管理、构建执行和错误处理。

### 主要功能特性
- **环境检查**：验证 Node.js 版本（要求 v18.19.1+ / v20.11.1+ / v22+）、npm 可用性
- **依赖管理**：自动安装 node_modules，处理 ajv 版本冲突问题
- **构建执行**：支持 build、dev、check 三种模式
- **错误处理**：完善的错误捕获与用户友好的错误信息
- **进度反馈**：详细的构建进度和状态输出

### 构建模式详解
- **build 模式**：执行 Web/PWA 生产构建，产物输出到 www/
- **dev 模式**：启动本地开发服务器，支持热重载
- **check 模式**：仅进行环境检查，不执行构建

### 配置参数
- **Configuration**：Angular 构建配置（默认 web_production）
  - 可用值：web_production | production | development
- **Yes**：自动确认（静默模式）

## Web/PWA 构建配置

### 配置集对比
项目提供了四种主要的构建配置集，针对不同的部署需求：

#### web_production 配置集
- **Base Href/Deploy URL**：/client/（用于 Web 部署）
- **预算限制**：初始包 2MB 警告，5MB 错误；组件样式 2KB 警告，4KB 错误
- **文件替换**：使用 Web 生产环境配置
- **输出哈希**：开启（提升缓存效率）

#### web 配置集
- **Base Href/Deploy URL**：/client/（用于 Web 开发）
- **文件替换**：使用 Web 开发环境配置
- **构建优化**：关闭（便于调试）
- **SourceMap**：开启（便于调试）
- **命名块**：开启（便于分析）

#### production 配置集
- **预算限制**：与 web_production 相同
- **文件替换**：使用原生生产环境配置
- **输出哈希**：开启

#### development 配置集
- **构建优化**：关闭（便于调试）
- **SourceMap**：开启（便于调试）
- **命名块**：开启（便于分析）
- **许可证提取**：关闭（减少构建时间）

### Web 版本特性
Web 版本配置具有以下特点：
- **webVersion 标识**：设置为 true，区分 Web 与原生应用
- **Base Href**：设置为 /client/，适用于 Web 服务器部署
- **部署 URL**：同样设置为 /client/，确保资源正确加载
- **开发友好**：关闭构建优化，便于开发调试

### PWA 特性集成
Web 构建配置集与 PWA 功能深度集成：
- **Service Worker**：启用并配置缓存策略
- **Web Manifest**：包含在构建资源中
- **缓存策略**：应用组预取，资源组懒加载
- **更新机制**：智能的资源更新与缓存管理

## 构建优化策略

### 代码分割与懒加载
- **路由级懒加载**：通过 Angular 路由配置实现按需加载
- **组件级懒加载**：大型组件按需加载，减少初始包大小
- **第三方库分离**：将常用第三方库单独打包，提升缓存效率

### Tree Shaking 优化
- **ES6 模块**：使用 ES6 模块语法，便于 Tree Shaking
- **无副作用模块**：标记无副作用的模块，允许完全移除
- **最小化导入**：避免导入整个库，仅导入需要的功能

### 资源优化
- **图片优化**：支持多种格式（SVG、PNG、JPEG、WebP）
- **字体优化**：Material Design Icons 和 Bootstrap 字体
- **CSS 优化**：SCSS 编译与压缩
- **JavaScript 优化**：TypeScript 编译与压缩

### 缓存策略
- **Service Worker**：实现智能缓存与离线访问
- **HTTP 缓存**：合理的 HTTP 缓存头设置
- **版本控制**：输出哈希确保缓存失效

### 性能监控
- **构建时间监控**：记录各阶段构建时间
- **包体分析**：定期分析包体组成，识别优化机会
- **运行时性能**：监控应用运行时性能指标

## 故障排查指南
- **构建后原生应用无法加载资源**
  - 检查 Capacitor 配置的 webDir 是否指向 www
  - 确认 angular.json 的 outputPath 与 Capacitor 的 webDir 一致
- **Service Worker 缓存未生效**
  - 检查 ngsw-config.json 的资源匹配是否覆盖所需文件
  - 确认构建时启用了 serviceWorker 并指定了 ngswConfigPath
- **环境变量未按预期替换**
  - 确认 angular.json 的 fileReplacements 正确映射到目标环境文件
  - 检查环境文件中的标识字段（如 production、webVersion）是否符合业务逻辑
- **包体超限或样式体积异常**
  - 查看 budgets 配置，定位初始包或组件样式阈值
  - 分析第三方库引入情况，必要时拆分或按需加载
- **调试困难或构建过慢**
  - development 配置适合调试；生产配置适合发布
  - 如需更快迭代，可在本地使用 development 配置并禁用 SourceMap 以外的优化
- **PowerShell 构建失败**
  - 检查 Node.js 版本是否满足要求（v18.19.1+ / v20.11.1+ / v22+）
  - 确认 npm 可用且网络连接正常
  - 检查 ajv 版本冲突问题，脚本会自动处理但可能需要手动干预
- **Web 构建路径问题**
  - 确认 baseHref 和 deployUrl 设置正确（/client/）
  - 检查 Web 服务器配置是否支持 SPA 路由
