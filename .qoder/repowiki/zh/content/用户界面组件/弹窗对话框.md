## 简介
本文件系统性梳理 Macro-Deck-Client-App 中的弹窗对话框组件，覆盖以下弹窗的职责、生命周期、模态控制、数据传递与用户交互：
- AddConnection：添加/编辑连接弹窗，支持手动输入与二维码快速设置
- Connecting：连接中弹窗，显示连接进度并支持取消
- ConnectionFailed：连接失败弹窗，展示错误详情
- ConnectionLost：连接丢失弹窗（首页模态）
- InsecureConnection：不安全连接提示弹窗（SSL 证书问题）
- SettingsModal：设置弹窗，集中管理应用配置
- **SendTextModal：发送文本弹窗，提供文本输入界面和两种发送模式选择，支持本地存储持久化**

同时说明弹窗与主页面的通信模式、事件流、数据回传与回调处理，以及在不同场景下的正确使用方法。

## 详细组件分析

### AddConnection（添加/编辑连接弹窗）
- 功能要点
  - 支持"快速设置"和"手动设置"两种入口
  - 原生平台下自动监听二维码扫描事件，解析二维码数据并触发网络接口扫描
  - 表单校验：主机/端口必填；确认后构造 Connection 对象并回传
  - 编辑模式：接收已有连接属性，允许修改并回传更新后的连接
- 生命周期与数据流
  - 初始化：订阅二维码扫描事件；若为原生平台且非编辑模式，自动尝试处理二维码
  - 快速设置流程：解析二维码 -> 打开网络接口扫描弹窗 -> 根据结果决定是否展示连接失败弹窗 -> 重置表单
  - 手动设置流程：直接填写表单 -> 校验 -> 构造 Connection -> 回传
- 与主页面通信
  - 主页面通过 ModalController.create 打开弹窗，传入 props（编辑模式或二维码数据）
  - onWillDismiss 监听确认事件，调用连接服务持久化数据

**更新** 端口输入字段的标签定位已从 floating 改为 fixed，提供更一致的标签定位体验

```mermaid
flowchart TD
Start(["进入 AddConnection"]) --> Mode{"是否为快速设置模式？"}
Mode --> |是| QR["监听二维码扫描事件"]
QR --> Parse["解析二维码数据"]
Parse --> Scan["打开网络接口扫描弹窗"]

Result --> |否| Fail["打开连接失败弹窗并重置表单"]

Validate --> |失败| Alert["显示错误提示"]
Confirm --> End(["结束"])
Alert --> End
```

### SendTextModal（发送文本弹窗）
- 功能要点
  - 提供文本输入界面，支持多行文本编辑
  - 两种发送模式：键盘输入（SEND_TEXT）和剪贴板粘贴（SEND_TEXT_CLIPBOARD）
  - 本地存储持久化：自动保存最后输入的文本，下次打开时自动恢复
  - 完整的国际化支持：标题、占位符、按钮文本均支持中英文切换
- 生命周期与数据流
  - 初始化：从 localStorage 读取上次输入的文本并填充到输入框
  - 发送流程：用户输入文本 -> 选择发送模式 -> 构建协议消息 -> 通过 WebSocket 发送 -> 保存到本地存储 -> 关闭弹窗
  - 取消流程：直接关闭弹窗，不保存任何数据
- 与主页面通信
  - DeckPage 通过 ModalController.create 打开弹窗
  - 弹窗内部直接与 WebSocketService 通信，无需返回数据给父页面
  - 发送完成后自动关闭弹窗

## 性能考量
- 并行网络检测：网络接口扫描采用 Promise.all 并行检测，缩短等待时间
- 超时与取消：HTTP 请求设置超时与销毁引用，避免内存泄漏与长时间阻塞
- 模态生命周期：及时取消订阅与停止扫描，防止后台资源占用
- 设置应用即时性：部分设置（唤醒锁、屏幕方向、主题）在保存时立即应用，减少切换成本
- **本地存储优化：SendTextModal 使用轻量级的 localStorage 进行文本持久化，避免频繁的网络请求**

## 故障排查指南
- 二维码扫描无响应
  - 检查摄像头权限是否授予
  - 确认扫描链接格式是否为 Macro Deck 快速设置链接
- 无可用网络接口
  - 确认服务器端口与 SSL 配置一致
  - 检查网络连通性与防火墙设置
- 连接失败弹窗频繁出现
  - 查看错误详情，核对主机、端口、SSL 与令牌
  - 尝试更换网络接口或调整服务器配置
- 设置保存后未生效
  - 检查平台差异（如 Android Oreo 对屏幕方向限制）
  - 确认相关服务已正确更新（唤醒锁、屏幕方向、主题）
- 国际化按钮文本显示异常
  - 检查 TranslatePipe 是否正确导入和使用
  - 确认 i18n 文件中是否存在对应的翻译键值
  - 验证 okText 和 cancelText 属性的绑定语法
- **文本发送功能异常**
  - 检查 WebSocket 连接状态是否正常
  - 确认协议消息格式是否符合服务端要求
  - 验证本地存储是否被浏览器安全策略阻止
  - 检查国际化文件中 sendText 相关键值是否正确配置

## 结论
弹窗体系围绕"模态展示 + 数据回传 + 事件联动"的设计展开，既保证了用户体验的一致性，又通过清晰的生命周期与数据流实现了高内聚低耦合。AddConnection 与 SettingsModal 分别承担"连接配置"和"应用配置"的核心职责，其余弹窗作为支撑组件完善了错误提示与流程引导。**新增的 SendTextModal 组件进一步扩展了应用的功能边界，提供了便捷的文本输入和发送能力，并通过本地存储实现了良好的用户体验连续性**。SettingsModal 组件的国际化增强和 SendTextModal 的完整国际化支持进一步提升了多语言环境下的用户体验，确保所有用户界面元素都能正确显示本地化文本。建议在扩展新弹窗时复用现有模式：明确 props 与回传数据、规范 onWillDismiss 处理、及时清理订阅与资源。
