# Macro Deck Client 需求与设计文档

## 项目概述

### 项目简介
Macro Deck Client 是一款跨平台的宏控制面板客户端应用，通过 WebSocket 与 Macro Deck 服务器通信，将移动设备转换为可编程的宏按钮控制面板。用户可以在服务器端配置各种宏命令按钮，通过客户端进行远程控制。

### 目标平台
- **原始平台**：Android、iOS、Web
- **目标平台**：HarmonyOS（本设计文档用于 HarmonyOS HAP 应用开发）

### 核心价值
- 将手机/平板转变为电脑的扩展控制面板
- 支持自定义按钮布局和宏命令
- 低延迟实时通信
- 多设备同时连接

## 一、功能需求

### 1.1 连接管理模块

#### 1.1.1 连接配置
| 功能项 | 描述 |
|--------|------|
| 添加连接 | 手动输入服务器 IP、端口、SSL 配置 |
| 快速设置 | 扫描二维码自动配置连接 |
| 编辑连接 | 修改已有连接的配置 |
| 删除连接 | 删除不需要的连接配置 |
| 排序连接 | 拖拽调整连接列表顺序 |
| 自动连接 | 支持连接可用时自动连接 |

#### 1.1.2 连接类型
| 类型 | 描述 |
|------|------|
| USB 连接 | 通过 USB 调试连接（127.0.0.1） |
| 无线连接 | 通过局域网 Wi-Fi 连接 |

#### 1.1.3 连接配置字段
```typescript
interface Connection {
  id: string;           // 唯一标识符
  name: string;         // 显示名称
  host: string;         // 服务器地址
  port: number;         // 端口号（默认 8191）
  ssl: boolean;        // 是否启用 SSL
  autoConnect: boolean;// 是否自动连接
  token?: string;      // 认证令牌（可选）
  index: number;       // 排序索引
}
```

### 1.2 WebSocket 通信模块

#### 1.2.1 连接建立
| 功能 | 描述 |
|------|------|
| 主动连接 | 用户点击连接按钮建立 WebSocket 连接 |
| 自动连接 | 检测到服务器可用时自动连接 |
| 连接确认 | 发送 CONNECTED 消息等待服务器接受 |

#### 1.2.2 消息协议（Protocol v2）
| 消息类型 | 方向 | 描述 |
|----------|------|------|
| CONNECTED | 客户端→服务器 | 连接确认，包含客户端 ID 和 Token |
| GET_CONFIG | 服务器→客户端 | 发送面板配置（行列数、间距、圆角等） |
| GET_BUTTONS | 客户端→服务器 | 请求按钮列表 |
| GET_BUTTONS | 服务器→客户端 | 发送完整按钮列表 |
| UPDATE_BUTTON | 服务器→客户端 | 更新单个按钮数据 |
| UPDATE_LABEL | 服务器→客户端 | 更新按钮标签文本 |
| BUTTON_PRESS | 客户端→服务器 | 按钮按下事件 |
| BUTTON_RELEASE | 客户端→服务器 | 按钮短按释放事件 |
| BUTTON_LONG_PRESS | 客户端→服务器 | 按钮长按事件 |
| BUTTON_LONG_PRESS_RELEASE | 客户端→服务器 | 按钮长按释放事件 |
| SEND_TEXT | 客户端→服务器 | 发送文本（键盘模式） |
| SEND_TEXT_CLIPBOARD | 客户端→服务器 | 发送文本（剪贴板模式） |

#### 1.2.3 连接状态管理
| 状态 | 描述 | 用户反馈 |
|------|------|----------|
| 连接中 | 显示加载动画和"正在连接..."提示 |
| 等待接受 | 显示"等待主机接受连接..."提示 |
| 已连接 | 进入面板页面 |
| 连接失败 | 显示失败原因和排查建议 |
| 连接断开 | 显示断开提示，支持自动重连 |
| 不安全连接 | SSL 证书验证失败提示 |

### 1.3 服务器发现模块（Ping 服务）

#### 1.3.1 网络扫描
| 功能 | 描述 |
|------|------|
| 定期 Ping | 每 1-1.5 秒检测一次服务器可用性 |
| HTTP Ping | 通过 /ping 端点检测 |
| 可用性通知 | 服务器变为可用/不可用时通知用户 |
| USB 检测 | 检测 USB 调试连接可用性 |

#### 1.3.2 Ping 端点
```
URL: {http|https}://{host}:{port}/ping
Method: GET
Response: JSON with ping timestamp
```

## 四、HarmonyOS 实现指南

### 4.1 技术栈选型

#### 4.1.1 推荐方案
- **开发语言**：ArkTS（TypeScript 超集）
- **UI 框架**：ArkUI
- **状态管理**：@ohos/axios + 简单状态模式
- **网络通信**：@ohos/axios（HTTP）、WebSocket API
- **本地存储**：Preferences API
- **国际化**：@ohos/i18n

### 4.2 模块映射

| 原始模块 | HarmonyOS 实现建议 |
|----------|-------------------|
| Angular 组件 | ArkUI @Component |
| Angular 服务 | ArkTS class 单例 |
| Ionic Storage | Preferences API |
| Capacitor 插件 | HarmonyOS 系统 API |
| WebSocket | 原生 WebSocket API |
| HTTP Client | @ohos/axios |
| 二维码扫描 | @ohos/@hms/corekit/Scan Kit |

### 4.3 权限需求

```json
{
  "requestPermissions": [
    {
      "name": "ohos.permission.INTERNET",
      "reason": "需要网络权限与服务器通信"
    },
    {
      "name": "ohos.permission.CAMERA",
      "reason": "需要相机权限扫描二维码"
    },
    {
      "name": "ohos.permission.GET_NETWORK_INFO",
      "reason": "需要获取网络信息用于连接配置"
    },
    {
      "name": "ohos.permission.KEEP_BACKGROUND_RUNNING",
      "reason": "需要后台运行保持连接"
    }
  ]
}
```

### 4.4 关键 API 映射

| 功能 | 原始实现 | HarmonyOS 实现 |
|------|----------|---------------|
| WebSocket | RxJS webSocket | import { WebSocket } from '@kit.NetworkKit' |
| HTTP 请求 | HttpClient | import { axios } from '@ohos/axios' |
| 本地存储 | Ionic Storage | import { preferences } from '@kit.ArkData' |
| 国际化 | ngx-translate | import { i18n } from '@kit.LocalizationKit' |
| 屏幕方向 | Screen Orientation | import { window } from '@kit.ArkUI' |
| 屏幕常亮 | KeepAwake | import { brightness } from '@kit.SystemBrightnessKit' |
| 二维码扫描 | Barcode Scanner | import { scan } from '@hms/corekit/Scan Kit' |
